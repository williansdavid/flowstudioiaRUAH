[11] DECISOES ARQUITETURAIS FIRMADAS NESTA SESSAO
------------------------------------------------------------------------
- DECISAO: Titulos SSR devem usar expressoes JSX com template
  string ({`texto ${var}`}) e NUNCA interpolacao JSX mista
  (texto {var}) dentro de <title>
  Justificativa: interpolacao JSX mista cria array de children
  no <title>, gerando warnings de hydration em SSR e
  potencialmente <title> duplicado
  Impacto: padrao agora obrigatorio para todas as rotas do
  projeto; deve ser checado durante a auditoria de head()

- DECISAO: Ordem de execucao das proximas tarefas:
  (1) Config por studio -> (2) Auditoria head() -> (3) Guards
  SSR -> (4) RLS Supabase -> (5) Landing publica
  Justificativa: cada passo e base do proximo; landing publica
  so pode ser exposta com guards + RLS solidos
  Impacto: define o roadmap das proximas ~2 semanas

- DECISAO: Investigar antes de planejar (regra reforcada)
  Justificativa: evitar planos baseados em suposicao; sempre
  validar API/arquivo real antes de propor solucao
  Impacto: arquiteto virtual deve sempre pedir conteudo de
  arquivo ou comando de inspecao antes de propor mudancas
