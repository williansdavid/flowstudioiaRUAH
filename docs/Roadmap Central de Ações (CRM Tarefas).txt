# Roadmap: Central de Ações (CRM / Tarefas)
**Objetivo:** Transformar o FlowStudio em um sistema ativo, gerando tarefas diárias automáticas para retenção e recuperação de clientes.

## Fase 1: Fundação e Backend (Data Fetching)
- [ ] Criar o arquivo de tipos `src/features/crm/types.ts` (definir `TaskType` e `TaskItem`).
- [ ] Criar a server function `src/features/crm/server/getDailyTasks.ts`.
- [ ] Implementar sub-query: **Pendências** (Agendamentos passados com status `pending` ou `confirmed`).
- [ ] Implementar sub-query: **Faltas** (Agendamentos com status `no_show` recentes para repescagem).
- [ ] Implementar sub-query: **Aniversariantes** (Clientes fazendo aniversário na semana/mês).
- [ ] Implementar sub-query: **Remarketing** (Clientes sem visitas há mais de X dias).
- [ ] Unificar e ordenar todas as sub-queries em um único array de retorno.

## Fase 2: Roteamento e Interface Base (Frontend)
- [ ] Criar a rota protegida `src/routes/_authed/admin/tarefas.tsx` no TanStack Router.
- [ ] Adicionar o item "Tarefas" ou "CRM" na navegação lateral (`src/features/admin-shell/config/nav-items.ts`).
- [ ] Criar o componente principal `src/features/crm/components/TasksPage.tsx`.
- [ ] Criar o componente de card `src/features/crm/components/TaskCard.tsx`.
- [ ] Implementar o layout visual (estilo Inbox/Feed) agrupado por prioridade ou tipo.
- [ ] Criar o `EmptyState` (tela de "Zero tarefas hoje, bom trabalho!").

## Fase 3: Ações e Interatividade
- [ ] Criar gerador de links do WhatsApp dinâmico (textos específicos para Aniversário, Remarketing, Cobrança de falta).
- [ ] Implementar Ação: **Resolver Pendência** (Mudar status do agendamento para concluído/PDV ou cancelado).
- [ ] Implementar Ação: **Dispensar/Concluir Tarefa** (Remover o card da tela após o contato).
- [ ] (Opcional/Futuro) Criar tabela `task_logs` no banco para registrar que a mensagem de remarketing já foi enviada, evitando cobrar o mesmo cliente duas vezes na mesma semana.

## Fase 4: Refinamento e Limites
- [ ] Aplicar limites de paginação/quantidade na query (ex: max 5 aniversários, max 10 remarketing por dia) para não sobrecarregar o usuário.
- [ ] Revisão de responsividade (Mobile-first).
- [ ] Testes de ponta a ponta no fluxo.
