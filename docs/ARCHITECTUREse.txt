# FlowStudio AI — Arquitetura do Sistema

Data: 23 de junho de 2026
Stack: TanStack Start (SSR) + Router + React Query + TypeScript + Supabase + Tailwind + Vite + Netlify

---

## 1. Visao Geral

Plataforma white-label para studios de beleza (saloes, barbearias, studios).
Cada studio tem deploy isolado: seu proprio Netlify, seu proprio Supabase, seu proprio banco, sua propria landing page e identidade visual.

### 1.1 Modelo de 2 Modulos

Modulo 1 — Sistema (nucleo compartilhado entre studios)
- src/lib/, src/routes/ estruturais, src/components/ core
- Auth SSR, supabase clients, query, utils genericos, guards, types do banco

Modulo 2 — Site (individual por studio)
- src/sites/<slug>/
  - config/ (branding, identity, content, businessHours, seo)
  - components/ (landing)
  - utils/ (fachadas)
  - types/
  - studio.ts (entry point)

Regra de ouro:
- src/lib/ nunca importa de src/sites/
- src/sites/ consome src/lib/core/ via fachadas (buildBrandingCss.ts, buildSeo.ts)

---

## 2. Stack (travada)

- Framework: TanStack Start (SSR)
- Roteamento: TanStack Router
- Estado servidor: TanStack React Query
- Linguagem: TypeScript (noUncheckedIndexedAccess: true)
- Banco: Supabase (Postgres + Auth + RLS)
- Estilizacao: Tailwind CSS
- Build/Bundler: Vite
- Deploy: Netlify (1 por studio)
- Icones: lucide-react
- Notificacoes: sonner
- Validacao: zod
- Animacoes: framer-motion

Proibido: microservicos, multi-tenant, dados compartilhados, Kubernetes, filas, CQRS, event sourcing.

## 11. Convencoes e Padroes

### 11.1 Feature-based Architecture
Cada feature em src/features/<feature>/ e auto-contida:
  <feature>/
    components/     Componentes React
    hooks.ts        Hooks + mutations/queries
    server/         Server functions (createServerFn)
    types.ts        Tipos da feature
    utils/          Utilitarios
    index.ts        Barrel publico (opcional)

### 11.2 Fluxo de trabalho (engenharia)
1. Investigar — validar codigo real / API
2. Confirmar — resumir entendimento
3. Propor — abordagem + impacto + riscos
4. Aguardar OK — so executar apos autorizacao
5. Entregar — codigo completo, tipado, loading/error/empty states, SSR-safe, mobile-first

### 11.3 Regras de UI
- Mobile-first — toda interface prioriza mobile
- Componentes com loading, error, empty states
- Modal com protecao contra perda de dados nao salvos (beforeunload + confirm)

### 11.4 Tipagem estrita
noUncheckedIndexedAccess: true no tsconfig — todo acesso a array exige verificacao de undefined.

## 12. Glossario

- Studio: Cliente da plataforma (ex: "Ruah")
- Staff: Profissional do studio
- TimeOff: Bloqueio de agenda (folga, almoco)
- WorkingHours: Grade semanal de horarios
- Bookable: Staff disponivel para agendamento
- Slot: Intervalo de tempo disponivel
