### 2. Definição de Interfaces e Tipos

As interfaces abaixo definem a estrutura de dados consumida pelo frontend para renderização da grade horária e cálculo de colisões visuais.

#### 2.1. Interface AppointmentItem
Representa o objeto de agendamento processado para a visualização.

```typescript
export interface AppointmentItem {
  id: string;
  starts_at: string;
  ends_at: string;
  status: 'pending' | 'confirmed' | 'completed' | 'cancelled' | 'no_show';
  client: {
    full_name: string;
    phone: string;
  };
  service: {
    name: string;
    duration_minutes: number;
    color?: string;
  };
  staff_id: string;
}
```

#### 2.2. Interface PositionedBlock
Define as propriedades geométricas para o posicionamento absoluto na grade da agenda (estilo Google Calendar).

```typescript
export interface PositionedBlock {
  id: string;
  top: number;      // Posição em pixels baseada na hora de início
  height: number;   // Altura em pixels baseada na duração
  width: number;    // Largura percentual para lidar com sobreposições
  left: number;     // Deslocamento horizontal para sobreposições
  data: AppointmentItem | StaffTimeOff;
}
```

### 3. Lógica de Busca (getDayAppointments)

A função getDayAppointments.ts é responsável por buscar todos os eventos de um dia específico, consolidando agendamentos e bloqueios de horário.


"A query deve filtrar por staff_id (opcional) e pelo intervalo entre 00:00:00 e 23:59:59 da data solicitada, convertendo para UTC antes da execução no Supabase."


1. **Input:** `date: Date`, `staffId?: string`.
2. **Operação:** Realiza um `SELECT` na tabela `appointments` com `JOIN` em `services` e `profiles`.
3. **Operação:** Realiza um `SELECT` na tabela `staff_time_off`.
4. **Output:** Retorna um objeto contendo dois arrays: `{ appointments: [], timeOff: [] }`.

### 4. Dependências do Projeto (package.json)

O projeto utiliza bibliotecas modernas para manipulação de UI e ícones, mantendo o bundle leve.

- **Framework:** React / Next.js
- **Ícones:** `lucide-react` (v0.453.0)
- **Estilização:** Tailwind CSS
- **Banco de Dados:** `@supabase/supabase-js`
- **Gerenciamento de Estado:** React Context / TanStack Query (recomendado para cache da agenda)

### 5. Regras de Arquitetura

O módulo segue o padrão Feature-based Architecture, garantindo que toda a lógica da agenda esteja isolada e protegida contra efeitos colaterais em outros módulos.

- **Localização:** Todo o código deve residir em `src/features/appointments`.
- **Componentização:** Componentes de UI (Calendar, Slot, EventCard) ficam em `components/`.
- **Lógica de Servidor:** Queries e Mutations ficam em `server/`.
- **Tipagem:** Definições globais da feature ficam em `types.ts`.
- **Isolamento:** Nenhuma outra feature deve importar componentes internos de `appointments`, exceto através de um ponto de entrada público (index.ts), se necessário.
